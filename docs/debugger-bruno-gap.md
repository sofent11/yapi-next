## Current Parity Matrix

| Area | Bruno OSS capability | YAPI Debugger status | Notes |
|---|---|---|---|

| Body modes | JSON, text, XML, GraphQL, SPARQL, file, form URL encoded, multipart | Partial | Schema/UI/runtime now accept XML, GraphQL, SPARQL, and file body for HTTP send. GraphQL query variables are materialized as JSON payloads, and the UI can fetch introspection, cache the schema summary on the request, clear stale cache entries, preserve that cache through Bruno JSON/OpenCollection JSON round-trips, generate query/mutation/subscription drafts from root fields with scalar, enum, list, and input-object variable placeholders, format variables JSON in-place, and save/load reusable operation snapshots on the request. | Request kinds | HTTP, GraphQL, gRPC, WebSocket, JS | Partial | Schema has `kind: http/graphql/grpc/websocket/script`; GraphQL sends over HTTP POST. Runtime fixtures now cover unary, server-streaming, client-streaming, and post-round-trip runnable assertions so regressions stay visible. Bidi streaming and broader session authoring polish are still pending. |
| Auth | Basic, bearer, API key, OAuth2, OAuth1, AWS v4, Digest, NTLM, WSSE | Partial | Schema/UI preserve all major Bruno auth families. Environment Center now also exposes an active-request precedence audit plus a searchable cross-scope catalog for project/environment/runtime/prompt layers so authors can spot shadowed names, request-linked gaps, and the right edit surface without changing runtime precedence. Script-time `pm.variables` now behaves as a dynamic scope resolver over collection/runtime variables, iteration data, environment vars, and in-memory globals while still writing back to the runtime variable store. ## Implementation Guardrails

- Do not vendor Bruno source into this repository.
- Keep YAPI Debugger's workspace format as the durable source of truth.
- Preserve existing YAPI-specific flows: import repair, browser capture, cases, collection reports, and Git helper.
- Any Bruno-parity item that is schema/UI-only must be documented as pending runtime support until it can send, import/export, and test successfully.
